Data Model for a Semantic Layer 
Back to the list of Data Models 
The purpose of a Semantic Layer is to provide a 'business-user-friendly' interface to the underlying Database or Data Warehouse

It replaces the names of tables and fields in the underlying Database by business terms.
For example, 'COUNT(Contract Deactivations) / COUNT(Customer)' is replaced by 'Churn Rate'.
