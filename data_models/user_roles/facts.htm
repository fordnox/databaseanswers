Detailed Requirement :-
Subject: Database design datamodel for User Roles
Question: Hi, I have to design a user interface with the following conditions. 
1. User Requests access, Approval Process, Revoking Process 
2. User can have 3 roles – Senior Manager, Project Manager, Consultant 
3. Each roles can have a role type of either “Editor” or “Reviewer” 
4. Project manager Editor can belong in multiple organizations, so does project Manager Reviewer. 
5. Consultant Editor can belong to multiple Programs, so does consultant reviewer. 

For Example the following scenarios should be satisfied. 
(1) User U1 requiring PM Editor access to program P1 
(2) User U2 requiring SM Reviewer Access 
(3) User U3 requiring Consultant Reviewer Access to Program P1 and Program P2. 
    Same user also requiring PM Editor access to Program P2 
(4) User U3 later requesting revoke access for Consultant Reveiwer for Program P1 and retaining the other roles
