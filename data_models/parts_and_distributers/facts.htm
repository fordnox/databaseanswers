The Area being Modelled is :-
Subject: Separate tables ?

Question: Should I create separate tables to hold the relationships found in the following: 

Module to Part A Module has 1-many Parts 

A Part can be found in 1-many Modules 

Module to Drawing 

A Module has 1-many Drawings 

A Drawing can be found in only 1 Module 

Part to Drawing 
A Part can be found in 1-many Drawings 
A Drawing can have 1-many Parts 

Part to Manufacturer 
A Part can have 1-many Manufacturers 
A Manufacturer can produce 1-many Parts 

Part to Distributer 
A Part can have 1-many distributers 

A Distributer can produce 1-many Parts 

Manufacturer to Contact 
A Manufacturer has 1-many Contacts 
A Contact belongs to only 1 Manufacturer 
Distributer to Contact 
A Distributer has 1-many Contacts 
A Contact belongs to only 1 Distributer Or should I create the 6 main tables 
and connect them through relationships? 

Right now I have tables named and some important componants: 
Modules: describing the module 
Parts: description, on hand, how many on hand 

DrawingsOfModules: file size, pages, file name/location, latest revision 

PartsInDrawings: Part, page it's on, and drawing revision Manufacturer: 
address... Distributer: address... Contact: phone/email info... 

PartsByManufacturer: PartID, ManufacturerID, Minimum Order Amount, Price, Lead time, comments 

PartsByDistributer: PartID, DistributerID, Minimum Order Amount, Price, Lead time, comments 

ManufacturerContact: Man-ID, Contact ID DistributerContact: Dis-ID, Contact ID 

Is this a good practice?
