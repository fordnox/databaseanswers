Glossary-driven User Scenario   
Here's a diagram showing the role of a Glossary-Driven User Scenario which contains six Steps. 
Step 1. The Vocabulary is presented to the User in accordance with the appropriate User Interface Design Guidelines.
Step 2. The User makes a request by specifying Search Terms.
Step 3. The Search terns are translated into a Web Service call in XML.
Step 4. The Web Service call is sent to the Information Management Platform (IMP), provided 'in the Clouds'.
Step 5. The IMP searches the Vocabulary and formulates the Response which is sent as a Web Service call back to the Database Answers Web Site.
Step 6. This Web Service Response is translated into an Internet Page and presented to the User.
