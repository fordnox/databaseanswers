Levels in a CRM Data Architecture
There are three Design Levels :-
  • A top-level view of the Data Architecture is defined as a Logical level, by a series of Data Islands, joined by Bridges.
  • The second level is a Physical Level, where the Data Islands generally correspond to Applications, and each Application will have its own Database Schema and Interface Schema.
  • The third level is an Implementation Level, where the Bridges become Data Access and Transfer mechanisms.
