Background to reviewing the Data Dictionary
The Data Dictionary is an essential component of the Data Architecture.
You can check out a Template for an Applications Inventory.

It is used to a number of key activities, incuding :-
  • Tracking progress
  • Generate Code for Data Migration
  • Resolving Data inconsistencies

    For example :-

    1. Inventory of Applications
      This Inventory of the Data Interface requirements for all Applications establishes software support needs.
    2. Documentation of Required Features
      This section of the Appendix makes it possible to identify the features of the Data Architecture which correspond to specific stated requirements in the source documents, such as the Functional Requirements.
    3. Glossary of Terms
      This section of the Appendix defines the Data Items in a Glossary of Terms.
      Note on Abbreviations :-
      D = Default value can be defined
      M= The Item is Mandatory
      V =Validation Criteria can be defined.
    4. List of Entities
      This section of the Appendix makes it possible to identify the features of the
      Data Architecture which are derived from the various Source Systems.
      The objective is to establish a WingCast-specific view of Data Entities and to avoid a vendor-specific approach.
      The initial implementation may be geared to particular vendors to facilitate progress.
